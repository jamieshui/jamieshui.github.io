# 从输入URL到页面呈现发生了什么？

主要可分为两大过程：

  • 加载过程
    1. DNS 解析：域名 -> IP 地址
    2. 浏览器根据 IP 地址（利用 TCP 协议通过三次握手）向 web 服务器发起 http 请求
    3. 服务器处理 http 请求，并返回给浏览器
  • 渲染过程
    1. 根据 HTML 代码生成 DOM Tree
    2. 根据 CSS 代码生成 CSSOM
    3. 将 DOM Tree 和 CSSOM 整合构造成 Render Tree
    4. 根据 Render Tree 渲染页面
    5. 遇到<script>则暂停渲染，优先加载并执行 JS 代码，完成后再继续
    6. 直至把 Render Tree 渲染完成

# 能不能说一说浏览器缓存?

浏览器缓存又分为强缓存和协商缓存：

  • 首先通过 Cache-Control 验证强缓存是否可用，如果强缓存可用，那么直接读取缓存；
  • 如果不可以，那么进入协商缓存阶段，发起HTTP请求，服务器通过请求头中是否带上 If-Modified-Since 和 If-None-Match 这些条件请求字段检查资源是否更新：
    • 若资源更新，那么返回资源和 200 状态码；
    • 若资源未更新，那么告诉浏览器直接使用缓存获取资源，并返回**304**状态码。

# 前端代码文件如何实现缓存管理？

  • 这里特指除了 HTML 外的代码文件，因为 HTML 文件一般不缓存或者缓存时间很短。

  • 一般来说，现在都会使用工具（比如Webpack）来打包代码，那么我们就可以对文件名进行哈希处理，只有当代码修改后才会生成新的文件名。基于此，我们就可以给代码文件设置缓存有效期，比如一年时间 Cache-Control: max-age=31536000，这样只有当 HTML 文件中引入的文件名发生了改变才会去下载最新的代码文件，否则就一直使用缓存。

# cookie，localStorage，sessionStorage，indexDB之间的区别？

特性\存储方式 cookie localStorage sessionStorage indexDB
数据生命周期 一般由服务器生成，可以设置过期时间 除非被清理，否则一直存在 页面关闭就清理 除非被清理，否则一直存在
数据存储大小 4K 5M 5M 无限
与服务端通信 每次都会携带在 header 中，影响请求性能 不参与 不参与 不参与
  • 从上表可知，cookie 已经不建议用于存储。如果没有大量数据存储需求的话，可以使用 localStorage 和 sessionStorage。对于不怎么改变的数据尽量使用 localStorage 存储，否则可以用 sessionStorage 存储。另外，对于 cookie 来说，我们还需要注意安全性，在HttpOnly为 false 的情况下，cookie信息能直接通过 JS 脚本来读取。
