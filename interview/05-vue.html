# 分阶段答出Vue生命周期钩子函数及每个阶段的职责

  1. 创建阶段： beforeCreate：实例刚从内存中创建出来，还没有初始化data和method created：实例已从内存中创建出来，此时初始化了data和method beforeMount：此时已完成模板的渲染，但还未渲染到界面中去 mounted：模板渲染到了浏览器，创建阶段结束，进入运行阶段
  2. 运行阶段 beforeUpdate：数据更新时停用，在更新前访问现有的DOM updated：页面重新渲染完毕，页面中的数据和data一致
  3. 销毁阶段 beforeDestroy：实例销毁前停用。在这一步，实例仍可用 destroyed：实例销毁后调用。该钩子被调用后，对应vue实例的所有指令被解绑，所有事件监听器被移除，所有子实例均被销毁。

# Vue父子组件生命周期调用顺序

  1. 加载渲染过程：父 beforeCreate -> 父 created -> 父 beforeMount -> 子 beforeCreate -> 子 created -> 子 beforeMount -> 子 mounted -> 父 mounted
  2. 更新过程：
    • 父组件：父 beforeUpdate -> 父 updated
    • 子组件：父 beforeUpdate -> 子 beforeUpdate -> 子 updated -> 父 updated
  3. 销毁过程：父 beforeDestroy -> 子 beforeDestroy -> 子 destroyed -> 父 destroyed

# Vue在哪个生命周期内调用异步请求？

  1. created / beforeMount / mounted
  2. 推荐在 created 钩子函数中调用异步请求：
    • 相对 beforeMount / mounted 能更快获取服务端数据，减少页面 loading 时间。
    • SSR（Server-Side Rendering 服务器端渲染）不支持 beforeMount / mounted 钩子函数，放在 created 中有助于一致性。

# 对keep-alive的理解

# keep-alive的定义

  • keep-alive是一个抽象组件，它自身不会渲染DOM元素，也不会出现在父组件中；使用 keep-alilve包裹动态组件时，它会缓存不活动的组件实例，而不是销毁它们。

# keep-alive的使用场景

  • 首页列表激活态：论坛贴吧新闻用于控制文章/邮件/通知的状态

# v-if/v-show的区别

  1. 相同点：都用于控制元素的显示/隐藏
  2. 不同点：
    • v-if ：首次渲染时——
      • 若条件为假，什么也不操作，则页面当作没有这个元素；
      • 若条件为真，则开始局部编译，动态地向 DOM 元素里添加元素；
      • 若条件从真变假，则也开始局部编译，删除这些元素。
    • v-show ：无论条件为真或假，第一次渲染都会编译出来，即将标签添加到 DOM 中，通过 display 属性来控制元素的显示/隐藏。
  • 性能角度：
    • v-show 的性能高于 v-if ,通过 display 属性来控制显示/隐藏仅会触发页面重绘； v-if 通过节点的增加删除控制元素是否显示会触发页面的回流；回流必会导致重绘。

# Vuex的定义

  • Vuex 是 Vue.js 的状态管理工具，用于解决vue中状态共享的问题。

# Vuex的使用场景

  • 在列表中增加/删除物品时，比如在电商购物车中增删商品，可用 Vuex 处理其数据流。

# Element-UI 组件库

  • Element-UI 是一个基于 Vue.js 的高质量组件库，提供了一整套实用的 UI 组件和工具，帮助开发者更快速、更简便地构建 Web 应用程序。Element-UI 具有丰富的组件，如表格、对话框、按钮、表单、导航菜单等，可以大大提高开发效率。
