# for-in 和 for-of 的区别

  • for - in ：ES5标准，遍历的是key（可遍历对象、数组或字符串的key）。

    for...in 不应该用于迭代一个关注索引顺序的Array。 for ... in 是为遍历对象属性而构建的，不建议与数组一起使用，数组可以用 Array.prototype.forEach() 和 for ... of 。

  • for - of ：ES6标准，遍历的是value（可遍历对象、数组或字符串的value）。

# == 与 === 的区别

  • 推荐使用 === 。
  • 若左右两侧的数据类型不一致：
    • == 会强转左侧操作数的数据类型，使其与右侧操作数的数据类型保持一致再进行比较数值是否相等。
    • === 直接返回 false 。
  • 特例：
    null == undefined // true  因为undeined的值派生自null的值
null === undefined // false
NaN == NaN // false   NaN（Not a Number）与任何数不相等，包括自身
NaN === NaN // false

# ... 扩展运算符（spread运算符）

  • 扩展运算符可以用于数组的构造，也可以用于调用函数时，将一个数组用作函数参数（即，将该数组转化为参数的列表，使其成为函数的参数）。

# Array 的常用函数

  • Array.from(arrayLike[, mapFn[, thisArg]]) ：常用于实现数组的浅拷贝。 Spread 运算符 ... 也可以实现数组的浅拷贝。
  • Array.prototype.includes(valueToFind[, fromIndex]) ：用于判断一个数组中是否包含一个指定的值，若包含则返回 true ，反之 false 。
  • Array.prototype.filter(callback(element[, index[, array]])[, thisArg]) ：过滤函数，传入某种方法对数组进行过滤。不改变原数组。
    • arr1.filter(item => arr2.includes(item)) ，其中 filter 函数利用 item => arr2.includes(item) 箭头函数表达式对arr1进行筛选。
  • Array.prototype.forEach(callback(currentValue [, index [, array]])[, thisArg])：遍历数组，并返回数组的每一项和索引。（类似于 for 循环遍历数组的每一项）

    Map 实例也可以使用 forEach 函数。 ATTENTION（踩坑记录）: forEach 中：不可使用 break 跳出循环且使用 return 无效。

  • Array.prototype.sort()：排序函数。会改变原数组。
    • arr.sort((a,b) => b - a); // 降序排列
